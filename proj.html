About The Project

Worked for an American global financial institution dealing with hedge fund.They are one of the largest alternative asset managers with more than US $ 25 billion in assets under management.My project dealt with writing code for automating the regression testing for interest rate swaps.The goal of the project was to automate the entire work flow right from creation of the trades in the trading system and verifying the cash flows andreference data for the trade.I used Ranorex (graphical user interface test automation framework for testing desktop, web and mobile application) and C#.

Current State Of The Project

In the current state the quality analysts had to manually create trades in the trade routing and management system and then they had to approve and post the trades. After this trade’s reference data (data which is static such as ISIN code) would be available in the reference data workstation. At the same time projected cash flows for the trade could be viewed in another application. The quality analysts had to verify whether trade data was available in reference data workstation and whether the project cash flows were made correctly. This had to done for different product types. More than 200 trades had to manually entered and their data had to be verified each time a new release was sent to production (application used by end users).

The Process

I started with automating the trade entry for all the product types.The trade entry also had many different variation one was the simple case in which you create a trade, another case was you unwind (sell it) the trade after creating it and finally you could assign your trade (your position) to someone else. After this I started with the validation part, the lookup for reference data was implemented but validating the cash flows were becoming difficult, as the application which was used to see the cash flows was a legacy application and as a result there were compatibility issues with Ranorex which was not able to work properly. As a result I was able to implementa few validations and could not implement all the validations.
